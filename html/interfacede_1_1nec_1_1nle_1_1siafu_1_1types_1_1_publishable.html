FlatData flatten ()
 

Detailed Description

All of the information managed by the simulation needs to implement this interface, which guarantees that whatever complex data type is used, it can be flattened into a String form and back from it.

The flattened String must contain all the necessary information to be rebuilt back again from the flattened form. Namely, information on the Publishable subtype, and all of the fields must be included. It is advisable that the flattened form is remotely human readable to ease usage outside of the simulator (for instance, in the CSV form).

It is mandatory that you implement a constructor of the type ClassName(FlatData), which will be used byFlatData to rebuild the Publishable object from itself. See FlatData for information on the formatting of the flattened string. reflection.

It is recommended that you override the toString method, since Siafu is likely to display it on GUI.
