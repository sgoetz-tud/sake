Detailed Description

This is the main class of the simulator. Upon running its main method, a new Controller is created. This, in turn starts three threads: Simulation: does the actual simulation and number crunching GUI: runs the user interface, if selected in the configuration file CommandListener: listens for external commands on the TCP port defined in the config file

Controller also acts as the reference point between the three threads, allowing them to share objects, such as the World or each other.

Asks the gui whether it wants to draw this iteration, and pauses until it wants to.

If the gui is not enabled in the configuration file, or if it chooses to skip this iteration, the method only action is to notify the GUI that it just missed an iteration. Note that the GUI might choose to skip iterationStep iterations to increase the interface's speed.
