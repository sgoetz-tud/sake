Detailed Description

The subclasses of SimulationData encapsulate the simulation information, including the agent, world and context models, the configuration file, the image maps and the sprites. They provide an easy interface to retreive all the information.

In order to get your SimulationData object, get an instance of the subclass by passing the path string to the getInstance method. Currently, only a folder or a jar file is supported. TODO:Document the jar/folder structure

Gets the names of the files from the simulation data associated to the given type.

For instance, calling getFilesOfType(FileType.OVERLAY) will return an ArrayList with the names of the overlays.

This method is similar to getFilesByPath, but does not actually open the files. Use wisely to avoid having too many open files at a time.
